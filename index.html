Chefの話

なぜChefを使う

普段開発するときにIDCFクラウド上でVMをたちあげて、一番最初に同じソフトウェアTmux, Gitなどをインストールする。 インストールするときのの依存パッケージを忘れたりするので、毎回同じ手順なら一箇所に管理したほうがいいんじゃない という考えでChefを使い始めました。

IDCFクラウド上のChefサーバー

Chefが管理しているサーバー


          $ knife node list
          chef-general-use    # 普段よく使われるサーバー
          chef-node-01        # テスト用のサーバー
          dev-riakcs2           # 開発用のRiakCSサーバー
          dev12-riakcs2       # dev12開発環境のRiakCSサーバー
          stg11-riakcs2       # stg11環境のRiakCSサーバー

Chefが管理しているRole


          $ knife role list
          setup                    # 全部のマシンに必要なレシピーはここに定義
          dev12-riakcs2      # dev12環境のRiakCSをインストールするためのレシピー
          stg11-riakcs2      # stg11環境のRiakCSをインストールするためのレシピー

Chefが管理しているCookbook


          $ knife cookbook list
          erlang                1.0.0
          general-setup   0.1.0
          riak                    0.1.0
          riak-cs               0.1.0
          riakcs2              0.1.0
          setup-vm           1.0.0
          stanchion          0.1.0

CookbookのソースはGithub上で管理

Cookbookの一例


          case node['platform_family']
          when 'rhel'
              execute "yum update" do
                  command "yum update -y"
              end
              node['setup-vm']['rpm-base-packages'].each do |pkg, command|
                  package "#{pkg}" do
                    action :install
                    not_if "which #{command}"
                  end
              end
              node['setup-vm']['rpm-dev-packages'].each do |pkg|
                  package "#{pkg}" do
                      action :install
                      not_if "rpm -qa | grep #{pkg}"
                  end
              end
          end
